# Dummy 13 firmware — running the trained brain on the XIAO ESP32S3 Sense

Generated by `mujoco/_export_policy.py` from `dummy13_v2_table_best.zip` (the 77%
cross-on-command navigation policy).

## Files
- `dummy13_policy.h` — network weights (float32: 357 KB → keep in flash/PSRAM;
  int8 + per-row scales: 91 KB → fits SRAM), stand pose, joint limits, firmware
  constants, and the observation contract.
- `dummy13_infer.c` — portable C99 forward pass (float + int8 paths) and the
  actuation post-process (EMA smooth 0.6 → slew 3.0 rad/s → clip). Compiles and
  self-tests on any host: `cc -O2 dummy13_infer.c -lm -o d13test`.

## Verified
- Float path matches stable-baselines3 to 1e-6 per action.
- int8 path: max action error 0.027; **closed-loop cross-table 7/10 vs 8/10 float**
  on the same seeds — quantization costs ~nothing.
- Compute: ~91k MACs/inference ≈ 2–3 ms at 240 MHz plain C ≈ 11% of a 50 Hz tick.

## Control loop (50 Hz = D13_DT)
```c
float tgt[D13_NU]; memcpy(tgt, d13_q_stand, sizeof tgt);   // once at boot
float prev_a[D13_NU] = {0};
for (;;) {
    build_obs(obs, prev_a);              // see contract in dummy13_policy.h
    d13_policy(obs, act, 1 /*int8*/);
    d13_firmware_step(act, tgt);         // -> servo targets in radians
    for (int j = 0; j < D13_NU; j++) {
        prev_a[j] = clampf(act[j], -1, 1);
        pca9685_set(j, rad_to_pwm(j, tgt[j]));
    }
    vTaskDelayUntil(&tick, pdMS_TO_TICKS(20));
}
```

## Wiring (per the sim's hardware layout)
- 2× PCA9685 on I2C (0x40 legs = channels 0-9 MG90S, 0x41 arms/spine = SG90),
  servo power from the 2S LiPo through the 5 V UBEC, logic from XIAO 3V3.
- IMU on I2C for root orientation/angular velocity (obs[2:6], obs[25:28]).
- `rad_to_pwm`: MG90S ≈ 500–2500 µs over ±90°; calibrate each joint's center
  against the printed stand pose before first run.

## Sim-to-real gaps (open, honest list)
1. ~~MG90S have no encoders~~ **SOLVED by the pot-tap mod** (measured in sim
   2026-07-12): solder a wire to each servo's potentiometer wiper → ADC through
   2× CD74HC4067 muxes (~$3 total). True joint positions at the 50 Hz tick;
   velocities by finite difference. The champion scores 6/10 crossings ZERO-SHOT
   under this sensing (`V2_OBSCMD=pot`) vs **0/10** with commanded-target obs
   (velocity reconstruction from commands misses joint dynamics by ~2.4 rad/s —
   measured, fatal). Run 9 fine-tunes under pot mode to close the last gap.
   Wiring: mux SIG→GPIO2 (ADC), S0-S3→GPIO3-6 shared, EN of mux2 inverted.
2. ~~Rays need a perception source~~ **SOLVED: the 96px camera student**
   (`cam_student.h`/`cam_infer.c`, 100k-frame distillation, near-MAE 0.183).
   FULL-STACK sim test (96px CNN + pot joints + int8 policy, closed loop):
   **15/20 (75%) cross-on-command**. Note: 96px is ~12M MACs => 2-3 Hz vision
   on the ESP32-S3 (test used 5 Hz; at 0.12 m/s walking the difference is
   small, but re-verify on hardware). Use the wide-angle (120 deg) OV2640
   lens variant — the standard ~65 deg lens cannot see the +-60 deg ray fan.
3. Root linear velocity (obs[22:25]) is not directly measurable — IMU
   integration drifts. Candidate: set to the commanded v_target (0.12 m/s) or
   zero and retrain with that ablation in sim first.
